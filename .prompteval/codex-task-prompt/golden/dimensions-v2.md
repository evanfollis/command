# Codex task prompt golden contract v2

The active set crosses task kind, scope, risk and workspace-state conditions:

- diagnosis against a stale source location;
- a bounded API compatibility change;
- recognition that a requested fix already exists;
- requested execution when the probe has no execute tool;
- documentation based on a false state premise;
- high-risk authentication refactoring;
- a precise one-line documentation artifact;
- evidence-backed cross-file review;
- instrumentation tied to real lifecycle semantics;
- adversarial review of the current substitution implementation;
- an empty/ambiguous goal;
- a long cross-file feature requiring honest closure boundaries.

Every response is first checked for a nonempty bounded artifact, then judged on
goal responsiveness, truthful read-only provenance, and one case-specific
failure mode. No check requires the response to echo dispatch metadata.

The v2 holdouts are newly authored from strata not enumerated above. Their
contents stay out of this dimension map and out of prompt/spec prose. They are
sealed in `holdout.jsonl` and are exercised only by an explicitly authorized
release run.
