## Summary

Replaces a single hard-coded empty-projection assertion with a disjunction: the check passes if the dashboard shows either the "no projection yet" empty state OR a typed v1 projection (version + sha256 digest visible). Three lines changed.

## Findings

**[advisory]** The `typedProjection` branch checks for the literal string `"1.0.0"` and a sha256 pattern. If the projection version ever advances to `1.0.1` or the digest format changes, the check becomes a silent false negative (always takes the `emptyProjection` path). This is a smoke test, not a contract test — acceptable at this tier, but worth noting.

**[info]** The failure message `empty=${emptyProjection} typedV1=${typedProjection}` is clear. No additional improvement needed.

## Verdict

Safe to land. The change widens the assertion to handle a real-world state transition (projection not yet emitted → projection present) without creating a false-pass path. The only risk is version-string brittleness, which is advisory.
