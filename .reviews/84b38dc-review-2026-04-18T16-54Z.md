## 1. XSS / injection in DOM-manipulating components

- `PortfolioCard.tsx:183-188` renders `CURRENT_STATE.md` via `ReactMarkdown` with `remarkGfm`. **No `rehypeRaw` plugin present** — raw HTML in markdown is stripped by default in ReactMarkdown v7+. Content source is server-side file reads. **No XSS vector.**
- `page.tsx:409-424` renders thread messages as plain React text nodes inside `<div className="whitespace-pre-wrap">`. No `dangerouslySetInnerHTML`. React escapes it.
- `sessions/[name]/page.tsx:101-106` renders tmux pane output inside `<pre>` as text only.
- No `dangerouslySetInnerHTML` or `rehypeRaw` usage found anywhere in `src/`.
- **Residual risk**: markdown link/image handling (clickable links in rendered CURRENT_STATE). Low risk — content is server-authored, single-user app.

**Triage: No action needed.** XSS surface is contained.

## 2. Auth gaps in thread and API routes

- Global JWT gate in `middleware.ts:4-36` protects everything except `/login`, `/api/auth`, and `/api/client-report`.
- Thread routes (`threads/route.ts`, `threads/[id]/route.ts`, `threads/[id]/messages/route.ts`) rely entirely on middleware — no in-handler `isAuthenticated()` call.
- Same middleware-only pattern on all mutating routes: `send`, `review`, `executive/ensure`, `executive/recover`.
- `/api/client-report` is intentionally public (telemetry beacon).
- `/api/auth` is intentionally unauthenticated (login endpoint).

**Triage: Accepted.** Middleware-only auth is the standard Next.js App Router pattern. Defense-in-depth (per-route checks) would be nice but is not a vulnerability — middleware runs before route handlers. Single-user app behind Cloudflare Tunnel further limits exposure.

## 3. State integrity under failure

- User message appended to transcript before CLI turn (`threadConversation.ts:207-209`). Timeout/crash leaves a lone user turn with no assistant response and no error marker.
- First-turn session creation can succeed externally but remain unsaved if the CLI call crashes before `updateThread()` runs (`226-233`).
- `appendTranscript()` does read-whole-file + rewrite (`57-63`), not atomic append. Crash during write can truncate. Malformed lines silently skipped by `getTranscript()` (`46-53`).
- Concurrency control is process-local `Map` (`194-200`). No cross-process coordination.
- Concurrent delete while a turn is running can leave orphaned transcript with no meta file.

**Triage: Accepted tradeoffs for single-user, single-process deployment.** These become real bugs if command ever runs multi-process. Already documented in CURRENT_STATE.md carry-forwards since 2026-04-17.
