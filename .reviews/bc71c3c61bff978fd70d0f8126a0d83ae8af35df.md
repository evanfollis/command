## Summary

Major structural commit introducing the prompt-eval governance infrastructure. Four prompts extracted from source (THREAD_OPENING_FRAME, buildReviewPrompt, buildCodexPrompt, buildOfflineSynthesisPrompt) into `src/prompts/`, wired through `src/lib/promptTemplate.ts`. Python adapters for prompteval use `render_via_runtime()` to call the real TS builders instead of re-implementing substitution. PROBE_PREFIX stripped of rubric-answering content. Input capture added via `src/lib/promptCapture.ts`.

## Findings

**[advisory]** `scripts/render-prompt.ts`: `id` is used as a filename component (`join(tmpDir, \`${id}.md\`)`) before the `switch(id)` validation. A crafted `id` could traverse out of `tmpDir`. Currently safe because (a) callers are eval adapters, not untrusted input, and (b) the `switch` throws for unknown ids before execution. **Follow-up commit should reorder: validate id before writeFileSync.** (See 2026-07-14 tick.)

**[advisory]** `src/lib/promptTemplate.ts`: Module-level `cache` and `PROMPT_DIR` are singletons. If multiple eval cases for the same `id` ran in the same process, the first candidate template would be returned for all subsequent cases. Currently safe because `render_via_runtime` spawns a subprocess per case, so each gets a fresh module. Non-issue for current architecture; worth noting if the harness ever moves to in-process rendering.

**[info]** `src/lib/promptCapture.ts` uses fire-and-forget append (no `await`). This is correct per the requirement that capture adds zero user-facing latency. An unhandled rejection from the write would be silent — acceptable for a telemetry path.

**[info]** PROBE_PREFIX is now behavior-neutral: describes the environment without instructing what to do. This is the correct fix for the self-answering grader bug.

**[info]** `fillTemplate` uses a replacer function, eliminating the `$&`/`$'`/`$$` corruption class from `String.replace(str, str)`. The comment documents why.

**[info]** Golden cases and holdouts are scoped correctly. `must_pass:true` only on cases where the judge can discriminate on substance (not harness limitations).

## Verdict

Safe to land with the advisory ordering note in `render-prompt.ts` treated as a follow-up (not a blocking deployment issue — the switch guards the execution path). The core design — real-builder rendering, behavior-neutral probe, separate capture path — is sound.
