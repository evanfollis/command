# Review: d013319 — Separate projection integrity from blocked research

**Date**: 2026-07-14T05-41Z  
**Reviewer**: Claude (Sonnet 4.6) — adversarial substitute (codex EROFS in tick session)  
**Change**: observatory.ts (+27 -5), observatory-test.ts (+13), CURRENT_STATE.md

## Summary

Splits research projection health into two independent signals: (1) "is the projection present and the record counts correct?" (contract integrity) and (2) "are any research records blocked?" This removes a coupling where a blocked-research state degraded the projection integrity signal rather than surface as its own signal.

Adds `researchHealth` struct to `PublicProjectionSummary` tracking active/blocked/completed/invalidated/withdrawn/superseded counts and block codes.

## Findings

**[advisory]** `collectKnowledgeState` now constructs reasons with inline count interpolation on every call. The strings reference `projection.researchHealth.blocked`, `projection.researchHealth.blockCodes`, etc. If `blockCodes` is an empty array (no blocked records but a blocked signal is present), the output is `"(block code missing)"` which is slightly misleading — should be `"(no block codes)"` or `"none"`. Minor UX issue.

**[advisory]** `sourceRef` carries absolute filesystem paths (`/opt/workspace/projects/synaplex/knowledge/projection.json`) through to the browser. Single-user auth-gated dashboard — not a security issue, but an unnecessary disclosure. See observatory-review-2026-07-14.

**[info]** `researchHealth` fallback values in `publicFallback` are all zeroes, which is a correct unknown state (not "0 blocked records known" but "unknown").

**[info]** `collectKnowledgeLoop` is still a single signal at this point. The separation of integrity from research-health is a sound design move.

## Verdict

Safe to land. The decoupling is sound — `projection present & counts verified` and `research blocked` are genuinely orthogonal signals that should not compound. The advisory issues are cosmetic and do not affect the auth boundary or data correctness.
