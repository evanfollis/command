Major commit: introduces the owner observatory as the new root page. `src/lib/observatory.ts` reads private workspace state (owner decisions, knowledge loop, automation health, telemetry, public projection, recent changes) and returns a typed `ObservatorySnapshot` to `ObservatoryDashboard.tsx`. ## Findings

**[advisory]** `observatory.ts` line ~59: `PRIVATE_KEYS` regex filters fields before snapshot JSON reaches the browser. The filter matches exact key names (word-boundary-free) — composite variants like `fileLocalPath` would bypass it. Low risk because the public projection schema is validated before filtering; arbitrary keys can't reach the surface without schema changes. But the filter's safety contract is weaker than it appears.

**[advisory]** `ObservatorySignal.sourceRef` carries absolute filesystem paths (`/opt/workspace/runtime/...`) through to the browser. Auth-gated, single-user dashboard — not a confidentiality breach, but unnecessary disclosure of server layout.

**[advisory]** `collectPublicProjection()` reads a JSON file from a configurable path. If the path is misconfigured or the file contains malformed JSON, `JSON.parse` throws and the observatory falls back via `safe()`. The `safe()` wrapper is correct; no crash propagation.

**[info]** `PRIVATE_KEYS` redaction runs after the typed projection schema check. This is the correct order.

**[info]** Browser smoke additions (+33 lines) test the observatory heading, decision queue, projection coherence, and truthful empty state. These are the right acceptance checks.

**[info]** All collectors run via `Promise.all()` with per-source `safe()` fallbacks and a 5-second timeout. Partial failures don't crash the snapshot.

**[info]** The package-lock.json churn is large (1846 lines) due to dependency updates. Not reviewed line-by-line; no known high-severity CVEs in packages at this version.

## Verdict

Safe to deploy. The observatory is auth-gated and uses the correct middleware chain. The two advisory items (PRIVATE_KEYS filter and sourceRef paths) are architectural notes, not blocking issues. The safe() fallback pattern means no observatory failure crashes the dashboard.
